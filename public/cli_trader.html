cli-trader

Filings analysis running LLM locally with easier SEC interfacing using their EDGAR API. Fast lookup of 10-K, 10-Q, 8-K etc. The goal is to combine a fast research interface with quick domain specific knowledge lookup using LLMs locally.

This was challenging, because the EDGAR API is quite obfuscated (which is why an entire second API exists thats built on the one from SEC and costs money to use) and secondly because the data cleaning process needs to be extensive to have useful input for the LLM.
